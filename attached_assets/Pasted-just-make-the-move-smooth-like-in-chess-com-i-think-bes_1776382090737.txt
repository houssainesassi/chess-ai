How it works:

Instant optimistic update — the moment you click or drop a piece, chess.js applies the move locally and the board re-renders immediately with the piece in its new position. No waiting for the server.

Sliding animation — an absolutely-positioned overlay piece renders at the source square and slides to the destination square in 150ms using a smooth cubic-bezier CSS transition (the same easing curve used by chess.com). During the slide, the destination square's piece is hidden so there's no visual doubling.

Parallel server call — the API call goes out at the same time as the local update. If the server confirms it (which it will for valid moves), the optimistic state is swapped for the real state seamlessly. If the server rejects the move (e.g. your turn ended), the board snaps back to the real position.

No double-moves — while a move is pending server confirmation, the board ignores clicks and drags, preventing race conditions.
